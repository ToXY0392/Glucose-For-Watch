## Root cause (logcat — X.4)

```
FATAL EXCEPTION: main
Process: com.widgetg7.mobile
java.lang.RuntimeException: Unable to create service
  com.widgetg7.mobile.sync.ActiveGlucoseSyncService
Caused by: android.app.ForegroundServiceStartNotAllowedException:
  Time limit already exhausted for foreground service type dataSync
  at android.app.Service.startForeground(Service.java:775)
  at com.widgetg7.mobile.sync.ActiveGlucoseSyncService.onCreate(ActiveGlucoseSyncService.kt:32)
```

**Explanation:** Android 14+ limits cumulative runtime for `dataSync` foreground services started from the background. When the quota is exhausted, `startForeground()` throws an **unhandled** exception in `onCreate()` → fatal crash ("has stopped").

**Possible triggers in code:**
- `ActiveGlucoseSyncController.start()` from `MainActivity`, `PhoneBootReceiver`, `PhoneAutoSyncReceiver`
- `PhoneWearRefreshRequestService` → `syncNow()` → service restart
- Redundant schedulers (alarm + WorkManager + service) restarting the FGS
